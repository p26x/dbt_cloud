#  grain_to_date: month
#1. Monthly Grain-to-Date (MTD)
# Logic: It sums data from the 1st day of the current month up to the current day.
#Behavior: On the 2nd of the month, it only shows 2 days of data. On the 30th, it shows 30 days.
 
 
# Window: 30 days
#2. Window of 1 Month (Rolling 30 Days)
#Logic: It sums data from today back to the same date last month (e.g., Feb 24 back to Jan 24).
#Behavior: The size of the "bucket" stays roughly the same every day. It never "resets" to zero.
     
 
#Setting,Logic,Real-World Use Case
#sum,Adds every daily value together,Total Revenue (MTD)
#average,Means the daily values,Avg Daily Bookings
#last,Takes the most recent value,Current Inventory
#min / max,Takes the lowest/highest value,Peak Daily Sales
 
 
 
#1. Filter group_by vs. Query group_by
#The group_by inside your YAML (filter): This only tells dbt the logic it should use to decide which rows to keep.
#It's like a subquery. It calculates: "Which listingids have > 11 bookings?"
#The Query group_by (Command Line): This tells dbt the structure of the table you want to see.
 
 
#metricflow
#This configuration is the "Shortcut Key" for your time-based metrics.
#By marking a specific dimension as primary_time: true,
#you are telling the dbt Semantic Layer that this is the official timeline for this model.
#1. It enables the metric_time shortcut
#If you have primary_time: true set on booking_date, you no longer have to type out the long, annoying entity prefixes in your terminal.
#Without it: dbt sl query --metrics ... --group-by bookingid__booking_date__day
#With it: dbt sl query --metrics ... --group-by metric_time__day
#dbt will automatically map metric_time to whatever column has that primary_time flag.
#2. It solves "Join Ambiguity"
#If your model has multiple dates (e.g., booking_date, created_at, updated_at), and you ask for a "Weekly Total," dbt won't know which date to use for the weekly reset.
#Setting primary_time: true tells dbt: "When in doubt, use this column to aggregate my numbers."
